✅ GARANTÍAS IMPLEMENTADAS

1️⃣  No hay textos fijos hardcodeados
   ❌ "Sistema POS Profesional"
   ❌ "LOS NILKAS"
   ❌ Slogans o nombres de empresa inventados
   ✅ SOLO datos desde Configuración del Negocio

2️⃣  PDF siempre fresco y actualizado
   ✅ Se regenera cada vez que se abre
   ✅ Cambios en Configuración se ven inmediatamente
   ✅ Sin reutilización de archivos en cache

3️⃣  Datos incompletos se manejan correctamente
   ✅ Si no existe telefono2, se omite (no muestra "—")
   ✅ Si no hay slogan, simplemente no aparece
   ✅ Si campo está vacío, no se muestra
   ✅ Nunca inventa datos

4️⃣  Single Source of Truth
   ✅ EmpresaService es la ÚNICA fuente
   ✅ Lee siempre desde BD (no cache)
   ✅ Fallback seguro a defaults si hay error

5️⃣  Compatibilidad backward
   ✅ Código antiguo sigue funcionando
   ✅ Parámetro 'business' es fallback
   ✅ Sin cambios necesarios en llamadas existentes

🎯 EJEMPLOS DE USO

EJEMPLO 1: Generar PDF dinámico
─────────────────────────────────

import 'core/services/empresa_service.dart';
import 'core/printing/quote_printer.dart';

final pdfData = await QuotePrinter.generatePdf(
  quote: quote,
  items: items,
  clientName: clientName,
  clientPhone: clientPhone,
  clientRnc: clientRnc,
  // NO necesitas pasar 'business' 
  // QuotePrinter usará EmpresaService automáticamente
  validDays: 15,
);

RESULTADO:
✅ PDF con nombre real de empresa desde Configuración
✅ Teléfono actual
✅ Dirección actual
✅ Sin textos fijos


EJEMPLO 2: Ver previa con datos frescos
────────────────────────────────────────

await QuotePrinter.showPreview(
  context: context,
  quote: quote,
  items: items,
  clientName: clientName,
  clientPhone: clientPhone,
  clientRnc: clientRnc,
  // Sin parámetro 'business'
  validDays: 15,
);

RESULTADO:
✅ Visor PDF abre PDF regenerado
✅ Cada apertura = PDF nuevo con datos frescos
✅ Cambios en Config se ven inmediatamente


EJEMPLO 3: Acceder a datos de empresa
──────────────────────────────────────

import 'core/services/empresa_service.dart';

final config = await EmpresaService.getEmpresaConfig();
print('Nombre: ${config.nombreEmpresa}');
print('Teléfono: ${config.getTelefono()}');
print('RNC: ${config.rnc}');

O métodos específicos:

final nombre = await EmpresaService.getEmpresaNombre();
final telefono = await EmpresaService.getEmpresaTelefono();
final direccion = await EmpresaService.getEmpresaDireccion();

RESULTADO:
✅ Datos siempre desde BD (sin cache)
✅ Fallback seguro a defaults

📋 CAMBIOS TÉCNICOS DETALLADOS

EmpresaService (NUEVO)
──────────────────────
Responsabilidades:
✅ Leer configuración actual desde BD
✅ NO cachear datos (siempre fresco)
✅ Proporcionar fallback seguro
✅ Ser la ÚNICA fuente de verdad

Métodos principales:
• getEmpresaConfig()     → EmpresaConfig completo
• getEmpresaNombre()     → String (nombre)
• getEmpresaTelefono()   → String? (teléfono)
• getEmpresaDireccion()  → String? (dirección)
• getEmpresaRnc()        → String? (RNC)
• getEmpresaEmail()      → String? (email)
• getEmpresaLogo()       → String? (ruta logo)


QuotePrinter (ACTUALIZADO)
────────────────────────────
Cambios en generatePdf():
  ANTES: Requería parámetro 'business'
  AHORA: Parámetro es opcional
  - Primero intenta EmpresaService
  - Si falla, usa parámetro 'business' como fallback
  - Si ambos fallan, usa defaults seguros

Cambios en showPreview():
  ANTES: Usaba nombre de archivo simple
  AHORA: Nombre único con timestamp
  - Format: cotizacion_{id}_{timestamp}.pdf
  - Garantiza regeneración en cada apertura
  - Evita cache de PDF viejo

Cambios en printQuote():
  ANTES: Podría reutilizar PDF
  AHORA: Regenera siempre
  - Datos siempre frescos
  - Cambios en Config aplicados inmediatamente
